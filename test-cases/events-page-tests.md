# Тест-кейси для сторінки Events (GreenCity)

### Test Case 1: Перевірка фільтрації подій за локацією (Львів)
**Preconditions:** Користувач знаходиться на сторінці https://www.greencity.cx.ua/#/greenCity/events

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | У блоці фільтрів знайти поле "Location" (або відповідний фільтр міст) | "Lviv" (Львів) | З'являється випадаючий список міст або можливість обрати Львів |
| 2 | Обрати "Львів" та застосувати фільтр | - | Список подій оновлюється. Відображаються лише ті картки подій, які проходять у Львові |

### Test Case 2: Перевірка переходу на сторінку деталей події
**Preconditions:** Користувач знаходиться на сторінці https://www.greencity.cx.ua/#/greenCity/events і бачить хоча б одну картку події.

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Навести курсор на картку будь-якої активної події | - | Картка візуально реагує на наведення (hover effect) |
| 2 | Клікнути на картку події (або кнопку "Details", якщо є) | - | Відбувається перехід на сторінку деталей обраної події. URL змінюється |
| 3 | Перевірити заголовок на відкритій сторінці | - | Заголовок сторінки відповідає назві обраної події з попереднього екрану |

### Test Case 3: Перевірка відображення дефолтного стану сторінки подій (UI)
**Preconditions:** Користувач відкриває браузер (без авторизації).

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Перейти за URL сторінки подій | https://www.greencity.cx.ua/#/greenCity/events | Сторінка успішно завантажується |
| 2 | Перевірити наявність основних елементів UI | - | Відображаються: заголовок "Events", панель фільтрів, сітка з картками подій (якщо вони є в базі), пагінація |

### Negative Test Case 4: Фільтрація подій за параметрами, яких не існує
**Preconditions:** Користувач знаходиться на сторінці https://www.greencity.cx.ua/#/greenCity/events

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | У блоці фільтрів обрати параметри, які гарантовано не мають жодної події (наприклад, неіснуюче місто у пошуку або специфічна комбінація тегів) | Локація: "TestCity999" | Система дозволяє ввести дані або обрати фільтри |
| 2 | Застосувати обрані фільтри | - | Сторінка не ламається (немає 500 помилки сервера). Відображається порожня сторінка замість списку карток з повідомленням про те, що подій не знайдено (наприклад, "No events found" або аналогічне) |

### Negative Test Case 5: Спроба створення події неавторизованим користувачем
**Preconditions:** Користувач **не авторизований** у системі. Відкрито сторінку https://www.greencity.cx.ua/#/greenCity/events

| Step | Action | Data | Expected Result |
| :--- | :--- | :--- | :--- |
| 1 | Знайти на сторінці кнопку створення нової події (наприклад, "Create event") та клікнути на неї | - | Система перехоплює дію. Замість переходу до форми створення події, з'являється модальне вікно або відбувається редирект на сторінку авторизації (Sign In / Sign Up) |
| 2 | Перевірити можливість обійти авторизацію | - | Створення події залишається заблокованим до моменту успішного входу в систему |
